# 🌐 DealUniverse — 営業自動化システム

> 面談前〜送信までを全自動化する、営業支援パイプライン

---

## 🧩 システム概要

営業面談の**準備→実施→分析→承認→送信→蓄積**をほぼ自動化するシステム。
人間が操作するのは最小限（1タップ・30秒承認など）にとどめる設計。

---

## 🛠 使用ツール一覧

| ツール | 役割 |
|--------|------|
| Google Calendar | 面談スケジュールの取得 |
| GAS（Google Apps Script） | 自動化エンジン全般 |
| Make（旧Integromat） | フロー制御・Webhook連携 |
| Claude API | AI分析・文章生成 |
| Google Contacts | 連絡先DB |
| Google Drive | 録音・議事録ファイル保存 |
| Google Sheets（CRM） | 面談情報・送信履歴の蓄積 |
| Google Chat | 社内通知チャネル |
| 連絡チャネル | Messenger / LINE / メール等 |
| GitHub Pages | Dashboardの可視化 |

## 🔄 フェーズ別フロー

### ① 面談前フェーズ（前日 or 当日：自動）
- スマホでカレンダーに面談登録
- 時間トリガーで当日面談を検索（GAS）
- 相手企業・人物名を調査（Claude API）
- ブリーフィング情報を生成
- 面談ブリーフシートをWebhook送信
- スマホに通知 → 読んで面談へ

---

### ② 面談中フェーズ（人間は録音1タップだけ）
- Googleレンズで名刺をスキャン（1タップ）
  - 住所・会社・電話・メールをGoogle連絡先へ自動登録
- 録音スタート（音声のみ、文字起こしはリアルタイムで自動実行）
- 録音終了 → 音声＋テキストが自動保存

---

### ③ 別れた直後フェーズ（名刺スキャン・30秒）
- 別れた後、その場でスマホを取り出してスキャン
- 新規連絡先をトリガーとして検知
- 名刺情報＋会社情報で連絡手段を分類・推奨
  - 例：経営者→スタートアップ、Messenger優先、メール副
- 推奨チャネルをCRMへ自動記録
- 推奨連絡手段をChatへ通知

---

### ④ 面談後分析フェーズ（全自動・並列）
- 録音テキストファイルを検知
- テキストデータを置いて並列処理を起動
  - 議事録・共創提案・感情分析を依頼
  - 出力：議事録要約 / 共創提案 / 感情トーン
- 面談情報をCRMへ即時記録
- 推奨チャネルに合わせた文面を生成
  - Messenger用・メール用・LINE用など、チャネル別・文体別を自動調整
- チャネル別フォロー文面を選択

---

### ⑤ 承認フェーズ（30秒で完結）
- 3点セット＋チャネル提案を通知
  - 議事録要約 / 共創提案 / 送信文面（チャネル別）
- スマホに通知
- 「OK」or「〜を修正して」と返信
- フィードバックを受信・学習ログへ追記

---

### ⑥ 送信フェーズ（心の仕事）
- 送信文面＋相手の連絡先URLを提示
- ここは人間が心を込めて「渡したこと」を感じて送る
- Messenger / LINE / メール / WhatsAppなどから**自分で送信**

---

### ⑦ 確定・蓄積・可視化
- 送信済みフラグ・最終データを記録
- Sheets API経由でデータを自動取得
- テーブルマップが更新される → トップ人物詳細・面談履歴
